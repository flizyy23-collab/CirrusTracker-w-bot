Changes made to WynnTrackerServer from the original


Bug fixes


report-raid-endpoint.js

Fixed a bug where looking up a player's UUID wasn't waiting for the response before continuing. This caused every raid report to fail with "Invalid player" for anyone who wasn't already saved in the database.

Before:
    if (!uuid) uuid = await requestUUID(player).uuid;

After:
    if (!uuid) uuid = (await requestUUID(player))?.uuid;


report-aspect-endpoint.js

Same bug as above but in the aspect reporting endpoint. Fixed the same way.

Before:
    if (!giverUUID) giverUUID = await requestUUID(giver).uuid;
    if (!receiverUUID) receiverUUID = await requestUUID(receiver).uuid;

After:
    if (!giverUUID) giverUUID = (await requestUUID(giver))?.uuid;
    if (!receiverUUID) receiverUUID = (await requestUUID(receiver))?.uuid;


wynn-api.js

The original code was sending an invalid placeholder value as an API token when talking to the Wynncraft API. This caused a 403 error which blocked all player authentication. Changed it so no token is sent, which works fine since the Wynncraft API is publicly accessible without one.

Before:
    const options = {
        url: `https://api.wynncraft.com/v3/player/${uuid}`,
        headers: {
            Authorization: `Bearer ${config.get("wynncraft-token")}`
        }
    };

After:
    const token = config.get("wynncraft-token");
    const options = {
        url: `https://api.wynncraft.com/v3/player/${uuid}`,
        headers: token && token !== "WYNNCRAFT_API_TOKEN" ? { Authorization: `Bearer ${token}` } : {}
    };


discord-bot.js

Added error handlers so the server no longer crashes when Discord or external services run into issues. Previously a single failed request could take the whole server down.

Added at the bottom of the file:
    client.on('error', (error) => {
        console.error('Discord client error:', error.message);
    });

    process.on('unhandledRejection', (error) => {
        console.error('Unhandled rejection:', error?.message);
    });


raids-command.js

The /raids command was crashing whenever crafatar.com (the service that provides player head images) was unavailable. Changed it so the command still works and just skips the thumbnail if crafatar is down.

Before:
    const response = await axios.get(`https://crafatar.com/renders/head/${uuid}?overlay=true`, { responseType: 'arraybuffer' });
    const buffer = Buffer.from(response.data, 'binary');
    const attachment = new AttachmentBuilder(buffer, { name: 'thumbnail.png' });
    ...
    .setThumbnail('attachment://thumbnail.png')
    ...
    await interaction.reply({ embeds: [exampleEmbed], files: [attachment] });

After:
    let attachment = null;
    try {
        const response = await axios.get(`https://crafatar.com/renders/head/${uuid}?overlay=true`, { responseType: 'arraybuffer', timeout: 5000 });
        const buffer = Buffer.from(response.data, 'binary');
        attachment = new AttachmentBuilder(buffer, { name: 'thumbnail.png' });
    } catch (e) {
        // crafatar unavailable, skip thumbnail
    }
    ...
    if (attachment) exampleEmbed.setThumbnail('attachment://thumbnail.png');
    await interaction.reply({ embeds: [exampleEmbed], files: attachment ? [attachment] : [] });


owed-command.js

The /owed command was crashing when there was no aspect data in the database yet. Added a proper message that says "No aspect data yet" instead of throwing an error.

Added before the pagination logic:
    if (fields.length === 0) {
        await interaction.reply({ embeds: [new EmbedBuilder().setColor(0x0099FF).setTitle('**Players with most Owed Aspects**').setDescription('No aspect data yet.')] });
        return;
    }


Customisation changes


database.js and aspects-command.js

Changed the owed aspects calculation from giving 1 aspect per 2 raids (whole numbers only) to 0.5 aspects per raid. This means players can now be shown as owed 0.5 or 1.5 aspects instead of only whole numbers. Also removed the floor so values can go negative (useful for tracking overpaid aspects).

Before:
    let owedAspects = Math.max(Math.floor(raids.length / 2) - totalAspects, 0);

After:
    let owedAspects = (raids.length * 0.5) - totalAspects;


New commands added


giveaspect-command.js (new file)

Adds a /giveaspect (player) (amount) command for admins. Records that a player has been given aspects, which reduces their owed count. Works in 0.5 increments.

Usage: /giveaspect Flizyy 1.5


setaspects-command.js (new file)

Adds a /setaspects (player) (amount) command for admins. Sets the owed aspects for a player to exactly the number you specify, regardless of their raid count. Useful for correcting mistakes.

Usage: /setaspects Flizyy 3.5 - this sets the player to be owed exactly 3.5 aspects


Config changes


guild-tag changed from TBGM to Crrs to match the actual Cirrus guild prefix as it appears on the Wynncraft website. The capitalization has to match exactly which is why it is Crrs and not CRRS.

sql host changed from 0.0.0.0 to localhost since the database is running on the same machine as the server.

Discord bot token and client ID filled in with the actual bot credentials.

MySQL password updated to match the one set during installation.
